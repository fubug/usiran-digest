# usiran-digest-check Skill

## 概述
usiran-digest-check是一个专门用于审核usiran-digest时事简报质量合规性的技能。该技能按照严格的新闻真实性、准确性和规范性标准对每小时生成的digest内容进行审计。

## 执行逻辑
1. **目标识别**: 获取当前时间前一个整点时段的digest文件
2. **文件读取**: 读取对应时段的.md格式的digest文件
3. **质量审核**: 按照以下标准进行逐项检查
4. **生成报告**: 生成详细的审计报告，标记违规项
5. **评分计算**: 根据违规严重程度计算质量评分

## 质量标准

### 必须合规项（违反即不合格）
1. **信源真实性**:
   - 所有引用的新闻链接必须真实存在
   - 链接必须指向实际报道内容
   - 不得引用不存在或错误的新闻源

2. **数据准确性**:
   - 所有数据必须来自可靠信源
   - 伤亡、时间等敏感数据必须有官方核实
   - 不得编造未经证实的数据

3. **人物真实性**:
   - 所有提及的人物必须真实存在
   - 官方任命和职位必须准确
   - 不得虚构官方人物和事件

### 格式规范项
1. **时间戳格式**: 必须使用ISO 8601 Z格式 (2026-04-26T17:00:00Z)
2. **前matter结构**: 必须包含id、date、title、tags、sources字段
3. **内容结构**: 中文摘要和English Summary必须完整
4. **标签准确性**: 标签必须准确反映内容主题

### 语言质量项
1. **客观性**: 避免主观臆断和情绪化表述
2. **专业性**: 使用专业、准确的新闻语言
3. **翻译质量**: 中英文内容必须准确对应
4. **时效性**: 时间窗口覆盖必须准确

## 检查流程

### 第一阶段：基础检查
- [ ] 文件存在性检查
- [ ] 前matter完整性检查
- [ ] 基础格式检查

### 第二阶段：内容质量检查
- [ ] 信源验证（HTTP 200状态码检查）
- [ ] 数据准确性验证
- [ ] 人物真实性验证
- [ ] 时间线一致性检查

### 第三阶段：规范性和语言检查
- [ ] 时间戳格式检查
- [ ] 内容时效性检查
- [ ] 翻译质量检查
- [ ] 标签准确性检查

## 违规等级划分

### 🔴 紧急违规（立即删除）
- 虚假信源（链接不存在）
- 虚构敏感数据（伤亡、军事数据）
- 虚构官方人物和事件
- 严重时间线错误

### ⚠️ 重要违规（需修正）
- 时间格式错误
- 内容时效性问题
- 数据准确性存疑
- 重大事实错误

### 🟡 轻微违规（需改进）
- 翻译质量问题
- 表述不够专业
- 标签选择不够精准
- 轻微格式偏差

## 质量评分标准

- **10/10**: 完美无违规
- **8-9/10**: 轻微违规，无需修改
- **6-7/10**: 中等违规，建议修正
- **4-5/10**: 较多违规，需重点修改
- **1-3/10**: 严重违规，可能删除
- **0/10**: 完全不合格，必须删除

## 输出格式

审计报告必须包含以下部分：
1. **基本信息**: 时间、目标、总体结果
2. **检查项列表**: 逐项检查结果
3. **详细违规分析**: 按严重程度分类
4. **质量评分**: 最终评分
5. **建议操作**: 具体修改建议
6. **历史对比**: 与历史标准的对比
7. **最终结论**: 是否合格及建议

## 执行频率
- 每小时执行一次
- 针对前一个整点时段的digest文件
- 在整点后10分钟内完成审计
