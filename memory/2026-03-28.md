## 🎯 **重要教训：记住自己的 Skill 能力**

### ❌ **错误案例**
- **时间**：2026-03-28 13:00-13:04
- **用户请求**：看腾讯新闻热榜
- **我的错误**：
  1. 直接尝试用 web API 获取（失败）
  2. 尝试 web_search 查找（无效）
  3. 绕了一大圈才想起有 `tencent-news` skill
  4. 浪费了 4 分钟时间和用户多次提醒

### ✅ **正确做法**
**遇到需求时，应该先检查 Skill 列表：**

1. **新闻类请求** → 使用 `tencent-news` skill
   - 热点新闻：`tencent-news-cli hot`
   - 早报：`tencent-news-cli morning`
   - 晚报：`tencent-news-cli evening`
   - AI 精选：`tencent-news-cli ai-daily`

2. **股票分析** → 使用 `stock-analysis` 或 `china-stock-analysis` skill

3. **天气查询** → 使用 `weather` skill

4. **视频处理** → 使用 `video-frames` 或 `video-generator` skill

5. **浏览器自动化** → 使用 `agent-browser` CLI（不是 OpenClaw browser 工具）

### 📋 **核心 Skill 清单**

| Skill 名称 | 功能 | 触发条件 |
|-----------|------|----------|
| tencent-news | 腾讯新闻热榜、早晚报、AI精选 | 用户要求看新闻、热点 |
| stock-analysis | 股票分析、投资组合 | 用户问股票、行情 |
| china-stock-analysis | A股分析、买卖建议 | 用户问中国股票 |
| weather | 天气预报 | 用户问天气 |
| video-frames | 视频截图、片段提取 | 用户处理视频 |
| video-generator | 文本转视频 | 用户生成视频 |
| agent-browser | 浏览器自动化 CLI | 需要浏览器操作 |
| daily-hub-summary | 每日信息汇总 | 自动化新闻汇总 |

### 🔄 **工作流程**

```
用户请求
  ↓
检查：我有相关的 skill 吗？
  ↓ 是
使用 skill 的 CLI 工具
  ↓
返回结果
  ↓ 否
考虑其他工具（web_search、web_fetch 等）
```

**而不是：**
```
用户请求 → 直接用熟悉的工具（如 web_search）
```

### 💡 **关键原则**

1. **Skill 优先**：先检查 skill 列表
2. **CLI 优于 API**：skill 的 CLI 通常比直接 API 更稳定
3. **记录工具路径**：知道每个 skill 的 CLI 在哪里
4. **环境变量**：注意 API Key 等配置
